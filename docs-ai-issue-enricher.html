AI Issue Enricher for Jira — Documentation

WSS Apps — Last updated: July 13, 2026

What it does

AI Issue Enricher turns vague Jira issues into actionable ones. From a panel on any issue, run an AI readiness check that scores the issue 0–100 against a Definition of Ready and lists exactly what is missing (no acceptance criteria, underspecified flows, unstated edge cases), each tagged high / medium / low severity. Then generate draft acceptance criteria in Given/When/Then form, edit or delete individual criteria, and apply them to the issue description with one click — the app appends an "Acceptance Criteria" section and never overwrites existing text. AI analysis runs exclusively on Atlassian's own LLM service — your issue content never leaves the Atlassian platform (Runs on Atlassian).

Installation

  1. Install the app from the Atlassian Marketplace on your Jira Cloud site.
  2. Open any issue, click the app actions button in the issue toolbar, and select AI Issue Enricher to add the panel.
  3. The panel stays on the issue view afterwards.

Using the app

1. Run the AI readiness check

  • Click Run AI readiness check. The app reads the issue summary and description and returns a readiness score (0–100), a verdict — READY, NEEDS DETAIL, or INCOMPLETE — a short summary, and up to six concrete gaps tagged by severity.
  • Typical gaps it finds: no acceptance criteria, missing user/goal, unspecified error handling, vague scope, undefined terms, missing reproduction steps.
  • If the issue is edited after a run, the panel shows a stale warning so you know to re-run.

2. Generate acceptance criteria

  • Click Generate acceptance criteria. The AI drafts 3–8 testable criteria in Given / When / Then form, covering the main flow plus key edge and failure cases.
  • Review the draft: Edit any criterion inline or Delete ones you don't want. Nothing touches the issue until you apply.

3. Apply to the issue description

  • Click Apply to issue description. The app appends an Acceptance Criteria heading and bullet list to the end of the description — existing description content is never modified or overwritten.
  • After applying, the draft is marked applied. Regenerate at any time to produce a fresh draft.

Data & privacy

The app stores only the AI analysis results and draft criteria (scores, verdicts, gap texts, criteria strings, timestamps) in Forge app storage, scoped to your site — no personal data is collected. Issue content is read via the Jira REST API within your site and processed exclusively by Atlassian's Forge LLM service; it never leaves the Atlassian platform. No data is used for AI training. See the Privacy Policy.

FAQ

Does the AI change my issues automatically? No. All AI output is a draft. The description is only modified when a person clicks "Apply to issue description", and even then content is appended — never overwritten.

What does the readiness score mean? 80+ = READY (a developer can pick it up cold), 50–79 = NEEDS DETAIL, below 50 = INCOMPLETE. The score is advisory — your team decides what "ready" means.

Which issue types does it work on? Any issue type with a summary and description — stories, tasks, bugs, epics.

What happens if the issue is edited after a check? The panel shows a staleness warning so you can re-run the analysis against the latest content.

Does it use my data to train AI models? No. Atlassian's Forge LLM service does not use your data for training.
