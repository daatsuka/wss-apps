AI Release Notes for Jira — Documentation

WSS Apps — Last updated: July 12, 2026

What it does

AI Release Notes turns the Jira issues in a fix version or any JQL query (e.g. a sprint) into polished, structured release notes in one click — written for the audience you choose: customer-facing, internal changelog, or technical / API. Drafts are grouped into New features, Improvements, and Bug fixes, generated by Atlassian's own LLM service, so your issue data never leaves the Atlassian platform (Runs on Atlassian).

Installation

  1. Install the app from the Atlassian Marketplace on your Jira Cloud site.
  2. Open any project and select AI Release Notes in the project sidebar (under Apps).

Using the app

1. Choose your source

  • Fix version — pick one of the project's versions. By default only issues whose status category is Done are included; switch to All in version to include everything.
  • Custom JQL — any query works, e.g. project = ABC AND sprint = 42 AND statusCategory = Done. Give the release a display name for the title.

2. Pick an audience

  • Customer-facing — benefit-oriented plain language, no issue keys or internal jargon; purely internal chores (CI, refactoring) are skipped automatically.
  • Internal changelog — one bullet per issue with its key, grouped by type, including an Internal/chores section.
  • Technical / API — issue keys kept, likely breaking changes surfaced first, and unknown upgrade steps marked [TO CONFIRM] instead of invented.

3. Control what goes in

  • Preview issues shows the exact list of issues that will be included before you generate.
  • Exclude labels — e.g. internal, no-changelog — removes matching issues from the notes.
  • Product name and extra context (e.g. "This release focuses on performance") shape the draft.

4. Generate, edit, and reuse

  • Click Generate release notes. The draft appears as editable markdown — review it, fix anything, then Save edits.
  • Every generated note is kept in a per-project History (last 50), so you can reopen, re-edit, or delete past notes.
  • Paste the markdown into Confluence, email, Slack, or your public changelog.

Data & privacy

The app reads issue summaries, types, statuses, and labels using the permissions of the person using it. Generated notes are stored in Forge app storage, scoped to your site. AI drafts are processed exclusively by Atlassian's Forge LLM service and never leave the Atlassian platform. No data is used for AI training. See the Privacy Policy.

FAQ

Can it invent features that aren't in my issues? The AI is instructed to use only the provided issues and to mark unknown details as [TO CONFIRM] rather than guessing. As with any AI draft, a quick human review is always recommended — that's why the output is editable before you publish it anywhere.

Does it work with sprints? Yes — use Custom JQL, e.g. sprint = 42 AND statusCategory = Done, and give the release a name.

How many issues can be included? Up to 100 issues per generation.

Who can use it? Anyone who can see the project. The app searches issues with the user's own permissions, so users only ever see issues they already have access to.

Does it use my data to train AI models? No. Atlassian's Forge LLM service does not use your data for training.
