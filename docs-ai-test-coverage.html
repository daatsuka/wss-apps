AI Test Coverage for Jira — Documentation

WSS Apps — Last updated: July 12, 2026

What it does

AI Test Coverage generates structured test cases from any Jira issue using Atlassian-hosted AI, tracks their execution status, and warns you when test cases go stale after the issue changes. All data stays inside your Atlassian site (Runs on Atlassian).

Demo

Installation

  1. Install the app from the Atlassian Marketplace on your Jira Cloud site.
  2. No configuration is required. The app adds a panel to every Jira issue.

Using the app

Generate test cases

  1. Open any Jira issue that has a summary (and ideally a description).
  2. Below the issue title, open Apps actions and select AI Test Coverage to show the panel.
  3. Click Generate test cases. The app sends the issue's summary and description to Atlassian's Forge LLM service and creates a set of test cases with preconditions, steps, and expected results.
  4. Each test case is categorized as functional, edge, or negative so you can see coverage balance at a glance.

Track execution

  • Click a test case's status badge to cycle it through Not Run → Pass → Fail → Blocked.
  • Record results step by step: each step has a pass/fail toggle, and the case status is derived automatically (any failed step → Fail, all steps passed → Pass).
  • Add a note / actual result to any case — useful for documenting failures.
  • Every case has a priority (P1/P2/P3). The panel header shows executed vs. total cases, unpassed P1 count, and a coverage summary by case type.
  • Click Complete test run to archive the current results into Run history (last 20 runs, expandable to per-case results with priorities, step counts, and notes) and reset all statuses for the next cycle.

AI gap analysis

Click AI gap analysis to have the AI review your existing test cases against the issue and propose only the missing scenarios (up to 5). Existing cases — including your edits and execution history — are never touched.

Export

Click Export CSV to get the full test case list (title, type, priority, status, steps, expected result, notes) for spreadsheets or external reports.

Edit and manage test cases

  • Each test case can be edited in place: title, type, steps, and expected result.
  • Delete cases you don't need, or click Add test case to write one manually.
  • Manually added cases are kept when you regenerate the AI set.

Stale test detection

If the issue is updated after test cases were generated, the panel shows a warning that the test cases may be outdated. Click Regenerate to create a fresh set based on the latest issue content.

Data & privacy

Test cases and execution status are stored in Forge app storage, scoped to your site. Issue text is processed only by Atlassian's Forge LLM service and never leaves the Atlassian platform. See the Privacy Policy.

FAQ

Which issue types are supported? All issue types that show the issue view panel (stories, tasks, bugs, etc.).

Can I edit generated test cases? Yes — every case can be edited, deleted, or added manually. Manual cases survive regeneration.

What's the difference between Regenerate and AI gap analysis? Regenerate replaces the AI-generated set (keeping manual cases and run history). Gap analysis keeps everything and only adds missing scenarios.

Does it use my data to train AI models? No. Atlassian's Forge LLM service does not use your data for training.
