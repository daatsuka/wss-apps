MCP Gateway: AI Agent Audit Log for Jira — Documentation

WSS Apps — Last updated: July 12, 2026

What it does

MCP Gateway records an audit trail of issue activity in your Jira site and flags actions performed by registered AI agent accounts (Claude, ChatGPT, or any MCP-connected tool). Admins get a filterable audit log, an AI agent registry, and CSV export — all stored inside your Atlassian site (Runs on Atlassian).

Installation

  1. Install the app from the Atlassian Marketplace on your Jira Cloud site.
  2. Go to Jira Settings → Apps → MCP Gateway to open the admin page.

Recommended setup

Because Jira events do not identify which client performed an action, the reliable governance pattern is one dedicated Atlassian account per AI agent:

  1. Create a dedicated Atlassian account for each AI agent (e.g. claude-agent@yourcompany.com).
  2. Configure your MCP client / AI tool to authenticate to Jira with that account.
  3. In the app's AI Agents tab, register that account using the user picker.

The built-in Setup Guide tab walks you through these steps.

Using the audit log

  • The Audit Log tab lists issue creates, updates, deletes, comments, attachments, worklogs, issue links, and project changes in reverse chronological order, with actor, timestamp, issue key, and field-level change details.
  • Events performed by registered AI agent accounts are flagged with an AI badge.
  • Click Export CSV to download the audit trail for compliance reviews.

Data & privacy

Audit records (actor account ID, timestamps, issue keys, change summaries) are stored in Forge app storage, scoped to your site, and visible only to site administrators. Nothing leaves the Atlassian platform. See the Privacy Policy.

FAQ

Can the app block AI agent actions? No. Forge apps receive events after they occur, so the app provides detection, audit, and alerting rather than prevention.

Does it audit actions outside issues? The app records issue create/update/delete, comment create/update/delete, attachment add/delete, worklog, issue link, and project create/update/delete events.

How long is audit data kept? For as long as the app is installed. Uninstalling removes stored data per Atlassian's Forge data lifecycle.
