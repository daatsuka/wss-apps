AI Document Approver for Confluence — Documentation

WSS Apps — Last updated: July 13, 2026

What it does

AI Document Approver adds a lightweight approval workflow to any Confluence page — request sign-off from one or more approvers, track decisions with notes and due dates, and keep a full activity log. Before you send a page for review, an optional AI quality check scores the page 0–100 and flags concrete issues (unfinished TODOs, vague statements, missing sections), so approvers receive documents that are actually ready. AI analysis runs exclusively on Atlassian's own LLM service — your page content never leaves the Atlassian platform (Runs on Atlassian).

Installation

  1. Install the app from the Atlassian Marketplace on your Confluence Cloud site.
  2. Open any page and click Approvals & AI Check in the byline (under the page title).
  3. The cross-space dashboard is available under Apps → Document Approvals.

Using the app

1. Run the AI quality check (optional but recommended)

  • Click Run AI quality check. The app reads the current page version and returns a score (0–100), a verdict — READY, MINOR FIXES, or NEEDS WORK — a three-line summary, and up to six specific issues tagged high / medium / low severity.
  • The report remembers which page version it reviewed. If the page is edited afterwards, the report is marked stale and you can re-run it with one click.

2. Request approval

  • Search approvers by name and add one or more of them.
  • Optionally add a note for the approvers and a due date (e.g. 2026-07-31).
  • Click Request approval. The page status becomes PENDING and each approver sees Approve / Reject buttons when they open the page.

3. Approve or reject

  • Each listed approver decides independently, with an optional decision note.
  • All approve → APPROVED. Any rejection → REJECTED immediately.
  • If the page is edited while a request is pending, a warning shows exactly which versions differ (e.g. "v3 to v5"), so approvers know they are reviewing newer content than requested.
  • The requester can cancel a pending request at any time. A new request can be started after any final decision.

4. Track everything in the dashboard

  • Waiting for me — pages where your decision is pending.
  • All requests — every request on the site with status, requester, approvers, AI score, due date (overdue dates are highlighted), and request time.
  • Export CSV — one row per request, ready for compliance or audit spreadsheets.

5. Activity log

Every event — AI checks with scores, requests, approvals, rejections, cancellations — is recorded per page with actor and timestamp. Click Show activity in the byline panel to review it.

Data & privacy

The app reads page content with the permissions of the person running the check — users can never analyze pages they cannot see. Approval records (page ID, approver account IDs and display names, statuses, notes, timestamps) are stored in Forge app storage, scoped to your site. AI analysis is processed exclusively by Atlassian's Forge LLM service and never leaves the Atlassian platform. No data is used for AI training. See the Privacy Policy.

FAQ

Does the AI decide approvals? No. The AI only provides an advisory quality score and issue list. Approval decisions are always made by the humans you choose.

What does the AI check look for? Completeness (TODO/TBD placeholders, missing sections), clarity, structure, and internal consistency. Each issue is tagged with a severity so authors know what to fix first.

Can approvers see pages they don't have access to? The byline panel lives on the page itself, so Confluence's own permissions apply — no access is ever widened by the app.

What happens if a page is edited during review? The request stays open, but both the panel and the AI report show clear version-mismatch warnings so nobody signs off on stale information unknowingly.

Does it use my data to train AI models? No. Atlassian's Forge LLM service does not use your data for training.
